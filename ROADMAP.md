# 研究アイデア創出エンジン ロードマップ

## 現在地

```
arXiv収集 → LLM構造化抽出 → SQLite/ChromaDB保存
    → 日次トレンド比較（今日 vs 過去7日）
    → 週次・月次合成レポート
    → メール通知（Gmail SMTP）
    → ローカル定期実行（launchd）
```

論文を「読んで整理する」段階。アイデアはまだ人間が生み出す必要がある。

---

## Phase 1 ｜基盤強化

**目標: 論文を「深く理解する」**

| 機能 | 内容 | 優先度 |
|---|---|---|
| トピック拡張 | RAG以外のテーマも追加収集（例: LLM agents, knowledge graphs） | 高 |
| クロストピック合成 | 複数トピックを横断して「共通課題」「借用できる手法」を検出 | 高 |
| 重要度スコアリング | 同じ課題を何本の論文が言及しているか数値化して優先表示 | 中 |
| ブートストラップ安定化 | arXiv レート制限対応・大量収集の安定化 | 高 |

---

## Phase 2 ｜アイデア生成

**目標: 論文の「ギャップ」から仮説を自動生成する**

### 処理フロー

```
論文群の limitations + open_questions
        ↓
「未解決問題リスト」の構造化・クラスタリング
        ↓
「手法A × 問題B の組み合わせ」を LLM が提案
        ↓
実現可能性・新規性スコアを付与
        ↓
研究アイデアレポートとしてメール送信（週次）
```

### 追加モジュール

- **`IdeaGenerator`** — 未解決問題と既存手法を組み合わせて仮説を生成
- **`NoveltyChecker`** — 提案アイデアがDB内の既存論文と重複しないか確認（ベクトル検索）
- **`IdeaStore`** — 生成したアイデアをDBに蓄積・バージョン管理

---

## Phase 3 ｜フィードバックループ

**目標: 使うほど精度が上がる**

- **興味度フィードバック**: メールに「👍 興味あり / 👎 興味なし」リンクを追加
- **プロファイル学習**: どのトピック・手法・アプローチに反応したかを記憶
- **パーソナライズ**: 自分の研究スタイルに合ったアイデアを優先表示

---

## Phase 4 ｜深い理解

**目標: アブストラクトを超えて論文本文を読む**

- **PDF全文解析**: arXivのPDFをダウンロードして手法節・実験節を読む
- **引用ネットワーク**: 引用元・引用先を追跡して重要論文を特定
- **パラダイムシフト検知**: 急激に注目を集めた手法を「ブレークスルー」として検出

---

## Phase 5 ｜自律エージェント化

**目標: 研究計画まで自動生成**

```
毎日の論文収集
    ↓
アイデア候補を自動生成
    ↓
「このアイデアはすでに研究されている？」を自律検索
    ↓
未着手と判断 → 研究計画（問い・手法・実験設計）を生成
    ↓
週次でアイデアポートフォリオとしてメール送信
```

---

## 実装優先順位

```
今すぐ  → Phase 1（トピック拡張・クロストピック合成・安定化）
次      → Phase 2（IdeaGenerator）
その後  → Phase 3（フィードバックループ）
長期    → Phase 4・5（PDF解析・自律エージェント）
```
